# Prototype Ordering System

A simple mobile-first ordering prototype for local businesses that do not need a full webshop.

The goal of this project is to create an easy-to-use ordering interface where customers can select products, choose quantities, review their order, enter contact details, and submit an order request. The order is not paid online. The business contacts the customer by phone to confirm the final details.

## Project Purpose

This project is built as a learning and practice project. It is designed as a template ordering system that can later be adapted to different small businesses, such as building material stores, firewood suppliers, local shops, or service providers.

## Current Features

* Mobile-first responsive layout
* Product list with quantities
* Cart calculation
* Net total, VAT, and gross total calculation
* Customer data form
* Order review page
* Final confirmation page
* Clear message that the order will be confirmed by phone
* Single-file HTML, CSS, and JavaScript prototype

## Planned Features

* HTML

Planned backend:

## Project Status

This project is currently in prototype stage.

The current version is suitable for demonstrating the ordering flow, testing the user experience, and collecting feedback before building a real backend system.

## Important Note

This is not a full webshop.
There is no online payment, no automatic invoicing, and no final order confirmation without human contact.

The system is designed to record order requests and support phone-based final confirmation.
